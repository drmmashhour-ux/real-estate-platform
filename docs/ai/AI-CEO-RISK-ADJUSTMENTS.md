# AI CEO — Risk-Driven System Adjustments

Strategic suggestions derived from **dispute prediction observability**, routed through **human approval** (`LecipmSystemBehaviorAdjustment`).

## Workflow

| Status | Meaning |
|--------|---------|
| `PROPOSED` | Draft recommendation (often generated by `ai-ceo-risk-adjustment.service.ts`) |
| `APPROVED` | Executive / admin accepted — still not auto-executed unless ops wires implementation |
| `REJECTED` | Declined with reason |
| `IMPLEMENTED` | Rolled out with optional `measuredOutcomeJson` |
| `REVERSED` | Rolled back — outcome captured |

## Services

| File | Purpose |
|------|---------|
| `ai-ceo-risk-adjustment.service.ts` | Draft bounded recommendations from dispute-risk trends |
| `ai-ceo-system-adjustment-policy.service.ts` | `validateBoundedAdjustmentText`, CRUD-ish status transitions |

## Guardrails

Heuristic keyword block on proposals containing punitive / guaranteed-return language.

**Never auto:**

- penalize users or brokers
- suppress listings without explicit policy + approval
- alter pricing / regulated investment flows without approvals

## Transparency

Every draft includes plain-language **explanation**, **confidence**, **urgency**, and **impact band** — aligned with existing `AiCeoRecommendationDraft` narratives.

# Internationalization (EN / FR / AR)

## Overview

The web app uses **English as default**, with **French** and **Arabic** as first-class UI locales. Messages live in `apps/web/locales/{en,fr,ar}.json` (flattened keys). Server and client resolve locale from cookie, `Accept-Language`, user preference, and optional launch flags.

## Key modules

| Area | Path |
|------|------|
| Locale tuple & types | `apps/web/lib/i18n/types.ts` (`UI_LOCALES`, `LocaleCode`) |
| RTL / LTR | `apps/web/lib/i18n/direction.ts` (`getDirection`) |
| Locale metadata (labels, BCP-47, RTL flag) | `apps/web/lib/i18n/locales.ts` (`UI_LOCALE_ENTRIES`) |
| Messages & `t()` | `apps/web/lib/i18n/messages.ts`, `I18nContext`, `server-translate` |
| Language switcher | `apps/web/components/i18n/LanguageSwitcher.tsx` |

## Persistence

- Cookie: `mi_locale` (see `LOCALE_COOKIE` in `lib/i18n/locales.ts`).
- Mirror: `localStorage.setItem("mi_locale", locale)` in `I18nContext` when the user switches language.

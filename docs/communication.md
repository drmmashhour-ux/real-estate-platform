# Communication rules

Default **async**; **meetings** only when they speed up decisions. Keeps execution fast across time zones.

---

## Daily updates

- **Format:** Short EOD post (see **`daily-execution.md`**): Done / Tomorrow / Blocked.  
- **Where:** One agreed channel (e.g. `#ship` or thread in team Slack).  
- **No** mandatory morning standup unless the team explicitly wants it.

---

## Weekly sync (45–60 min)

**Agenda (fixed order)**

1. **Metrics** (10 min) — **`performance.md`** snapshot; one insight.  
2. **Shipped** (10 min) — Demos optional; links to PRs/releases.  
3. **Priorities** (15 min) — Top 3 for next week; owner per item.  
4. **Risks** (10 min) — Technical, legal, market; single next step each.  
5. **Hiring / ops** (5 min) — If relevant.

**Output:** Written summary in wiki or Notion; link in Slack.

---

## Async rules

| Rule | Why |
|------|-----|
| **Prefer thread** over DMs for decisions that affect others | Searchable, inclusive |
| **Deadline + owner** in requests | Reduces ping-pong |
| **Document conclusions** after long threads | Avoid re-debating |
| **Respect focus hours** — urgent = call; else async | Deep work |

---

## Escalation

1. **Peer** — Try resolving in thread or 15 min call.  
2. **Lead / Founder** — Scope, priority, or cross-team conflict.  
3. **Incident** — Follow **`incident-response.md`** if production is broken.

---

## Tools (pick and stick)

- **Chat:** Slack / Discord  
- **Tasks:** GitHub Projects / Linear / Notion  
- **Docs:** Repo `docs/` + one external page for sensitive HR only

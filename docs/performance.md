# Team performance (execution metrics)

**Note:** This doc is about **people, product, and business** outcomes. For **application performance** (latency, bundles, DB), see **`performance-optimization.md`**.

---

## What we track

Roll up **weekly** in the weekly sync; **monthly** for trends. Use a simple spreadsheet or your analytics stack—perfection is optional; **consistency** matters.

| Area | Metric | Owner |
|------|--------|--------|
| **Product** | Features merged to `develop` / `main` | Eng lead / Founder |
| **Quality** | P1/P2 bugs opened vs closed; incident count | Eng |
| **Users** | Signups, MAU/WAU, activation % (define one core “aha” event) | Growth + Founder |
| **Revenue** | GMV, take rate, MRR, or pipeline $ (pick stage-appropriate) | Ops/sales + Founder |
| **Growth** | Leads, CPL, content output, top channel | Growth |

---

## Minimum viable dashboard

1. **Engineering:** PR throughput + open P1 bugs.  
2. **Growth:** Weekly signups + one funnel step (e.g. started booking).  
3. **Revenue / pipeline:** Weekly total + stage movement.  

---

## How to use it

- **No vanity:** If a metric doesn’t drive a decision, drop it.  
- **One owner per metric** — avoids “everyone’s responsible = no one.”  
- Tie **`roles.md`** success metrics to this sheet quarterly.

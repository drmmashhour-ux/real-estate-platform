# Differentiation from existing tools

**Purpose:** Technical contrast for counsel and product—not comparative advertising.

## Generic CRMs / listing backends

- Typically store listings and workflows; they do **not** ship a **unified** trust + deal + fraud **decision object** with **confidence-separated** presentation and **issue-coded** next actions tied to the same evidence model.

## Spreadsheet or static calculators

- Often assume complete inputs and emit a single number without **explicit confidence**, **escalation**, or **cross-listing** fraud signals.

## Marketplace “featured” or popularity scores

- Usually reflect engagement or fees paid, not **evidence-normalized** trust and **risk-adjusted** deal quality.

## Appraisal or brokerage CMA tools

- Focused on **valuation** with professional standards; this system’s deal layer is **decision support** for operators and sellers, with disclaimers and separate confidence.

## This system (summary)

- **Multi-engine** output (trust, deal, fraud) with **shared evidence**.
- **Confidence-aware** recommendations and UI.
- **Deterministic** issue codes feeding **next actions**.
- **Calibration** artifacts (human vs model) for continuous improvement.

# Growth execution

Daily and weekly **growth** rhythm aligned with **`roles.md`** (Growth) and **`tasks.md`** (`growth/*`).

---

## Daily

| Block | Action |
|-------|--------|
| **Morning** | Check yesterday’s numbers (signups, spend, top landing); note one anomaly. |
| **Content** | Ship **one** meaningful unit: post, email, LP tweak, or ad variant (don’t zero the day). |
| **Ads** (if live) | Budget caps, creative fatigue, geo/device breakdown; pause losers. |
| **Leads** | Route inbound to **`sales-execution.md`** playbook; log source in CRM. |

---

## Weekly

- **One experiment** start or finish: hypothesis → change → metric → learning (one paragraph in doc or Notion).  
- **Content calendar** for next week (topics tied to cities, hosts, or seasonal hooks).  
- **Attribution sanity:** UTM discipline on new links; fix broken tracking.  

---

## Platform promotion

- **BNHub / LECIPM:** Highlight trust, local inventory, host value prop—rotate angles (see `docs/outreach/` if you use it).  
- **Social proof:** Reviews, host quotes, metrics you’re allowed to share.  
- **Partners:** Co-marketing only when **`sales-execution.md`** has a signed path.

---

## Guardrails

- No misleading claims; align with legal **`docs/`** and product reality.  
- Staging URLs for unreleased features in **internal** tests only.

---

## Metrics (tie to `performance.md`)

- Signups, CPL, activation %, channel mix.  
- Weekly: one slide or table for **`communication.md`** sync.

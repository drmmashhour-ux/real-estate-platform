## Boundary notes

- The repo currently has two module trees: `apps/web/modules` and `apps/web/src/modules`. Treat `apps/web/modules` as domain/application services and `apps/web/src/modules` as feature UI plus newer vertical modules until an ADR replaces this split.
- DB access should enter through `@/lib/db` for app runtime code. Direct `new PrismaClient()` is acceptable in CLI scripts, seeds, and the realtime sidecar only when documented.
- Direct `process.env` reads remain common. New production-critical code should prefer the env/flag modules and log degraded behavior without exposing secrets.
- API handlers should return stable public errors and log internals through structured logging or `safeApiError`.
- Feature flags remain centralized in `apps/web/config/feature-flags.ts` and `apps/web/lib/launch/resolve-launch-flags.ts`.

# Daily execution system

A simple rhythm so the team ships every day without endless meetings. Works for **remote** and **hybrid**.

---

## Morning (15–30 min)

1. **Check tasks** — Board (GitHub Projects, Linear, Notion—pick one tool and stick to it).
2. **Scan comms** — Slack/email for blockers only; defer deep reading to a later slot.
3. **Prioritize** — Write **one** “must finish today” outcome (not a task list of 10).
4. **Align** — If you depend on someone, send **one** short message with what you need and by when.

---

## Work block (core hours)

- **Engineering / design:** Deep work on the single priority; PRs ≤ ~400 lines when possible; ship behind flags if risky (**`release-strategy.md`**).
- **Growth:** Execute **`growth-execution.md`** (content + ads + measurement).
- **Ops / sales:** Execute **`sales-execution.md`** (pipeline + onboarding).

**Rules**

- Default to **async**; meetings only when sync saves time.
- Block **90–120 min** focus windows; batch Slack twice daily if it helps.

---

## End of day (10–15 min)

1. **Report progress** — In your team channel (or tool), post:
   - **Done:** 1–3 bullets  
   - **Tomorrow:** top priority  
   - **Blocked:** optional, with owner tag
2. **Update board** — Move tasks; link PRs or docs.
3. **Handoff** — If you’re OOO tomorrow, leave one line for who picks up what.

---

## Weekly anchor

- Use **`communication.md`** for the **weekly sync** agenda (outcomes, metrics, risks).
- Founder (or lead) restates **top 3** company priorities every Monday in writing.

# Role responsibilities (BNHub / LECIPM)

Per-role **mission**, **daily tasks**, **weekly goals**, and **success metrics**. Keep this lightweight: review quarterly, adjust as the company stage changes.

---

## Founder

| | |
|--|--|
| **Mission** | Align the team on a winning strategy; allocate focus, budget, and talent so BNHub reaches product–market fit and sustainable revenue. |
| **Daily** | Scan metrics dashboard; clear 1–2 decisions blocking ICs; 1:1s or async check on critical path. |
| **Weekly** | Prioritize top 3 outcomes; review pipeline (hosts, brokers, revenue); hiring funnel. |
| **Metrics** | Revenue / MRR or GMV; runway; activation of core journey; team velocity vs plan; NPS or qualitative feedback trend. |

---

## Full-stack engineer

| | |
|--|--|
| **Mission** | Ship reliable product on web and shared services; keep the monorepo maintainable and fast. |
| **Daily** | Pick highest-priority task from board; PR small increments; fix CI/lint; respond to review requests. |
| **Weekly** | Close agreed features/fixes; reduce tech debt in one chosen area; document one non-obvious behavior if needed. |
| **Metrics** | Features/fixes **merged**; PR cycle time; incident count/severity; **build:ci** health; p95 API latency (if tracked). |

---

## Mobile engineer

| | |
|--|--|
| **Mission** | Deliver a stable mobile experience aligned with BNHub’s core guest/host flows. |
| **Daily** | Implement or fix mobile issues; sync with API changes; test on device. |
| **Weekly** | Release or internal build to staging; parity checklist vs web for 1–2 critical flows. |
| **Metrics** | Crash-free sessions; store review trend; time to ship mobile fixes; alignment with **`release-strategy.md`**. |

---

## UI/UX designer

| | |
|--|--|
| **Mission** | Make complex real-estate and booking flows feel obvious; lift conversion and trust. |
| **Daily** | Deep work on active screens; respond to engineering questions; update Figma/specs. |
| **Weekly** | Usability pass on staging; 1–2 shipped improvements or specs handed off; component/pattern doc update. |
| **Metrics** | Task completion rates (funnel steps); qualitative session feedback; design debt backlog size. |

---

## Growth / marketing

| | |
|--|--|
| **Mission** | Grow qualified demand (guests, hosts, brokers) with measurable channels. |
| **Daily** | Content or ad ops; check dashboards; reply to inbound; log experiments. |
| **Weekly** | Ship content calendar; review CAC/channel; one experiment concluded with learnings. |
| **Metrics** | Signups, qualified leads, cost per lead, content reach, conversion landing → signup. |

---

## Operations / sales

| | |
|--|--|
| **Mission** | Convert interest into live listings and revenue; keep operators and partners successful. |
| **Daily** | Outreach, demos, onboarding steps; support tickets; CRM updates. |
| **Weekly** | Pipeline review; N hosts/listings activated; partnership follow-ups; feedback to product. |
| **Metrics** | Meetings held; deals closed or advanced; time-to-first-listing; churn/cancellation reasons. |

---

## Cross-functional expectations

- Everyone uses **`communication.md`** for cadence and **`tasks.md`** for naming.
- Engineering follows **`team-workflow.md`** and **`git-workflow.md`**.
- Releases follow **`release-strategy.md`**.

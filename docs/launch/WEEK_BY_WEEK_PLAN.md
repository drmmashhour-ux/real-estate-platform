# Week-by-week plan (first month)

Realistic, **execution-focused** cadence. Adjust dates to your calendar; keep the **sequence** (demo ready → outreach → learn → widen).

---

## Week 1 — Prepare

| Task | Done when |
|------|-----------|
| Finalize **demo flow** end-to-end (script + clicks) — see [DEMO_PROCESS](./DEMO_PROCESS.md) | You can run it twice without errors |
| **Landing page** ready: what it is, who it’s for, clear CTA (book demo / request access) | Link you’re proud to send |
| Draft **3 outreach variants** (short DMs / emails); personalize per person | Saved in notes or doc |
| Build a list of **30–50 targets** (brokers, small teams, warm contacts) | Names + channel (LinkedIn, email, phone) |
| **Internal demo rehearsal** (co-founder or friend plays skeptic) | Notes on confusing spots |

**Weekly outcome:** You can invite someone to a demo with confidence.

---

## Week 2 — Conversations

| Task | Done when |
|------|-----------|
| Send **10–20 messages/day** on weekdays (see [LAUNCH_PLAN](./LAUNCH_PLAN.md) outreach section) | Log sent + follow-ups |
| **Book demo calls**; protect slots on your calendar | At least several calls scheduled |
| Run **first 5–10 demos** using the script | Each logged in [FEEDBACK_LOOP](./FEEDBACK_LOOP.md) |
| **Collect feedback** after every call | 3–5 questions every time |

**Weekly outcome:** Real reactions, not guesses.

---

## Week 3 — Learn and onboard

| Task | Done when |
|------|-----------|
| **Refine product** from top themes (UX, bugs, missing piece) | Shipped or scheduled fixes |
| **Fix friction** on signup / first action (see onboarding in [LAUNCH_PLAN](./LAUNCH_PLAN.md)) | Retest with one user |
| **Onboard first real users** (accounts, short setup help) | They complete first meaningful action |
| **Track usage manually** — see [METRICS_TRACKING](./METRICS_TRACKING.md) | Simple sheet updated |

**Weekly outcome:** A small set of people actually *using* the product.

---

## Week 4 — Widen carefully

| Task | Done when |
|------|-----------|
| **Increase outreach** slightly only if Week 2–3 didn’t break support | Still 10–20/day unless you can handle more |
| Ask happy users for **intros** (“Who else should see this?”) | 2+ introductions requested |
| **Improve onboarding** copy/tooltips based on repeated confusion | Fewer repeated questions |
| **Testimonials** only if genuine (quote, LinkedIn, short video) | Optional; no pressure |

**Weekly outcome:** Referrals starting *or* clear learnings on why not.

---

## After week 4

- Move to [ITERATION_PLAN](./ITERATION_PLAN.md) as your default rhythm.  
- Re-read metrics weekly ([METRICS_TRACKING](./METRICS_TRACKING.md)).  
- Expand audience only when **“get real usage before scaling”** is satisfied for your current cohort.

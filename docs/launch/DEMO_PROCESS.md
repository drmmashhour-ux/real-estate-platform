# Demo process (repeatable)

**Target length:** ~15–25 minutes total (including Q&A). **Stay in control:** follow the script; don’t wander the UI.

---

## Agenda

| Block | Time | Purpose |
|-------|------|---------|
| **1. Intro** | ~1 min | Who you are, why you’re talking, no pitch deck unless they love slides. |
| **2. Problem framing** | ~1 min | Their world: missed follow-ups, scattered docs, offer chaos—**ask one question** so they feel heard. |
| **3. Product demo** | ~5 min | Linear path through **one core story** (e.g. client → listing context → offer / doc / message—pick what’s strongest today). |
| **4. Questions** | ~5–10 min | Discovery + objections; see [FEEDBACK_LOOP](./FEEDBACK_LOOP.md). |

If time runs short, **cut optional branches**, not the core story.

---

## Before the call

- [ ] **Demo account** prepared: known login, seeded data that matches the story (1–2 clients, 1 listing or deal context).  
- [ ] **Browser**: logged in, zoom 100%, notifications muted.  
- [ ] **Reset** demo data if previous demos polluted state (use your staging/demo reset procedure—same path every time).  
- [ ] **Backup:** one Loom or screenshot flow if live demo fails.

---

## Demo account preparation

- Use **staging or demo tenant** if you have one; never risk production customer data.  
- Pre-create: user, **tenant/workspace**, sample client, sample listing/deal, **one** notification or message so the product feels alive.  
- **Document** login steps in a private note so anyone on the team can run the same demo.

---

## Reset demo data

- After heavy exploratory demos, **reset** to the baseline seed so the next call starts clean.  
- Automate if possible (script or admin button); if manual, use a **checklist** so nothing is forgotten.

---

## What NOT to show (unless they ask)

- Half-built settings, empty admin panels, experimental toggles.  
- Long lists with no data—use seeded samples or a tight filter.  
- Anything that might **break** during the call without a workaround.

---

## Adapt by user type

| Type | Emphasize |
|------|-----------|
| Solo broker | Speed, fewer clicks, mobile if relevant. |
| Small team | Handoffs, visibility, “who did what.” |
| Skeptical / busy | One job-to-be-done only; offer to email a 2-minute Loom after. |

---

## Rules

1. **Do not click randomly**—narrate each step: “I’m opening the client record because…”  
2. **One primary path** per demo; “we can also…” only if time and interest.  
3. **Pause** after the hero moment; let them react before moving on.  
4. End with a **clear next step**: trial access, second call, or intro to a colleague.

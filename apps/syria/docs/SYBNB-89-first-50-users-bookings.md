# ORDER SYBNB-89 — First 50 users + 10 bookings

Operational playbook for **Hadiah Link** (`apps/syria`): acquire early supply and travelers/stayers, run manual concierge bookings until traction proves repeatable.

Related context (no duplication required): [`GROWTH-EXECUTION-SYRIA.md`](./GROWTH-EXECUTION-SYRIA.md), [`LAUNCH-STRATEGY-SYRIA.md`](./LAUNCH-STRATEGY-SYRIA.md).

---

## 1. Supply first (before heavy outreach)

| Gate | Minimum | Geography focus |
|------|-----------|-----------------|
| Listings | **30** | Damascus + Latakia (titles/areas/tags aligned so browse/search feels credible) |
| Hotels | **5** | Same regions |

Pause broad paid/boost campaigns until these gates are met so replies stay **supply‑matched**.

---

## 2. Daily outbound rhythm

Track each day in a simple sheet (date → counts).

| Channel | Daily target |
|---------|----------------|
| WhatsApp / DMs | **30** messages |
| Facebook housing groups | **3** posts (vary wording; follow group rules) |
| Hotel / owner outreach | **10** contacts (cold intro → listing/on‑platform booking promise) |

---

## 3. First 50 users — sources checklist

Count anyone **meaningfully contacted or onboarded** (posted listing, completed signup intent, or received concierge intake):

- [ ] Friends & family referrals  
- [ ] Facebook housing groups (Damascus / Latakia / Syria diaspora where relevant)  
- [ ] Hotel WhatsApp lines  
- [ ] Sellers sharing links to their Hadiah Link listings  

**Target:** **50** such contacts/onboards logged.

---

## 4. Booking conversion flow

For **every** interested traveler/stayer, capture **before** sending inventory:

1. **City**  
2. **Dates** (check‑in / check‑out or range)  
3. **Number of guests**  
4. **Budget** (currency + rough band)

Then pick **only 1–2 best‑fit listings** (trust > breadth).

---

## 5. Closing scripts (Arabic)

### Opening + qualification

```text
مرحبا 👋
شو المدينة والتاريخ وعدد الأشخاص؟
ببعتلك أفضل الخيارات المتاحة.
```

### If they favor one option — invite booking request

```text
ممتاز 👍
إذا حابب، فينا نرسل طلب حجز الآن ونأكد مع المالك.
```

Adapt tone to the chat channel (WhatsApp vs. Messenger voice stays conversational).

---

## 6. First 10 booking requests — operations discipline

Until automation confidence is high, treat each booking as **manual**:

| Step | Owner |
|------|--------|
| Request logged on‑platform or via concierge ticket | Ops |
| **Immediately** ping seller (call/WhatsApp) | Ops |
| Confirm availability, rules, price, deposit expectation **before** promising guest | Ops |
| Mark booking **confirmed in-product only after seller approval** | Ops |

**Personally manage** each of the first **10** booking paths—notes template helps audit (`guest summary → seller confirmation quote → guest acceptance → confirmed flag`).

---

## Acceptance criteria (definition of done)

| Metric | Target |
|--------|--------|
| Real users **contacted or onboarded** | ≥ **50** (logged with date/source) |
| Booking **requests created** | ≥ **10** |
| **Manually confirmed** bookings (seller-approved path complete) | ≥ **3** |
| Supply gates | ≥ **30** listings incl. focus regions + ≥ **5** hotels |
